# تحديث الدومين - إصلاح القنوات المباشرة

## 📋 المشكلة
بعد تغيير الدومين إلى `www.amlive.shop`، القنوات المباشرة لا تعمل ويظهر فقط علامة التحميل.

## 🔍 السبب
التطبيق كان يستخدم IP مباشر للسيرفر السحابي:
```typescript
const CLOUD_URL = "http://62.171.153.204:8090";
```

هذا يسبب مشاكل:
- ❌ مشاكل CORS (Cross-Origin)
- ❌ Mixed Content (HTTPS/HTTP)
- ❌ البورت 8090 قد يكون محجوب

## ✅ الحل المطبق

### 1. تحديث ملف `web-app/src/constants/api.ts`
```typescript
// قبل
const CLOUD_URL = "http://62.171.153.204:8090";

// بعد
const CLOUD_URL = "https://www.amlive.shop";
```

### 2. تحديث ملف `web-app/next.config.js`
```javascript
// قبل
const CLOUD_URL = 'http://62.171.153.204:8090';

// بعد
const CLOUD_URL = 'https://www.amlive.shop';
```

## 📝 ملاحظات مهمة

### إعدادات Nginx المطلوبة
تأكد من أن Nginx يوجه الطلبات بشكل صحيح:

```nginx
server {
    listen 80;
    listen 443 ssl http2;
    server_name www.amlive.shop;

    # تطبيق الويب (Next.js)
    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # السيرفر السحابي (البث)
    location /hls/ {
        proxy_pass http://localhost:8090;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_buffering off;
        add_header Access-Control-Allow-Origin *;
    }

    location /api/xtream/ {
        proxy_pass http://localhost:8090;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
    }
}
```

### التحقق من عمل التحديث
1. افتح الموقع: `https://www.amlive.shop`
2. اذهب إلى القنوات المباشرة
3. جرب تشغيل أي قناة
4. يجب أن تعمل القناة بدون مشاكل

### إذا لم تعمل القنوات

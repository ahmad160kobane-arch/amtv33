# 🔍 تشخيص مشكلة القنوات

## 🐛 المشكلة المكتشفة:

القنوات المضافة يدوياً **موجودة في Backend** لكن **لا تظهر في التطبيق** لأن:

1. ✅ Backend API يحتوي على القناة:
   ```json
   {
     "id": "29acd253-358e-414f-b309-c6c8c284b911",
     "name": "Abalfadhl TV",
     "group": "عام",
     "enabled": 1
   }
   ```

2. ❌ السيرفر السحابي يعرض فقط قنوات Xtream (IPTV):
   ```
   /api/xtream/channels → قنوات IPTV فقط
   ```

3. ❌ التطبيق يستخدم `/api/xtream/channels` بدلاً من `/api/channels`

---

## 🔧 الحل:

### الخيار 1: تعديل التطبيق (موصى به)

تغيير التطبيق ليستخدم `/api/channels` من Backend مباشرة بدلاً من `/api/xtream/channels` من السيرفر السحابي.

**المزايا:**
- ✅ يعرض كل القنوات (IPTV + يدوية)
- ✅ لا حاجة لتعديل السيرفر السحابي
- ✅ أبسط وأسرع

**العيوب:**
- ⚠️ يحتاج تحديث التطبيق (Mobile + Web)

---

### الخيار 2: دمج القنوات في السيرفر السحابي

تعديل `/api/xtream/channels` ليدمج القنوات من Backend.

**المزايا:**
- ✅ لا حاجة لتعديل التطبيق
- ✅ يعمل مع الإصدار الحالي

**العيوب:**
- ⚠️ أكثر تعقيداً
- ⚠️ يحتاج تعديل السيرفر السحابي

---

## 📝 التطبيق الصحيح:

### 1. في Mobile App:

**الملف:** `mobile-app/constants/Api.ts` أو `mobile-app/services/api.ts`

**قبل:**
```typescript
const CHANNELS_URL = `${CLOUD_SERVER}/api/xtream/channels`;
```

**بعد:**
```typescript
const CHANNELS_URL = `${API_BASE_URL}/api/channels`;
```

---

### 2. في Web App:

**الملف:** `web-app/src/services/api.ts` أو مشابه

**قبل:**
```typescript
const response = await fetch(`${CLOUD_SERVER}/api/xtream/channels`);
```

**بعد:**
```typescript
const response = await fetch(`${BACKEND_URL}/api/channels`);
```

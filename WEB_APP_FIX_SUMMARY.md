# إصلاح مشكلة البث في تطبيق الويب

## المشكلة
تطبيق الويب كان يعرض علامة "جارٍ التحميل..." بشكل مستمر ولا يشغل الفيديو.

## السبب
1. السيرفر (`cloud-server`) يعيد مسار نسبي: `/proxy/live/123/index.m3u8`
2. تطبيق الويب على Railway يعمل على HTTPS
3. المتصفح لا يستطيع تحميل المسار النسبي لأنه يحتاج URL كامل
4. الكود القديم كان يحاول تحويل HTTP إلى مسار نسبي، لكن المسار كان نسبي أصلاً!

## الحل
تعديل `web-app/src/constants/api.ts` في دالة `requestFreeStream`:

```typescript
// إذا كان المسار نسبي (يبدأ بـ /)، حوّله إلى URL كامل للـ VPS
if (streamUrl.startsWith('/') && typeof window !== 'undefined') {
  streamUrl = 'http://62.171.153.204:8090' + streamUrl;
}
```

## النتيجة
- تطبيق الويب يتصل مباشرة بالـ VPS: `http://62.171.153.204:8090/proxy/live/123/index.m3u8`
- CORS مفعّل في السيرفر (`app.use(cors())`)
- البث يعمل بسلاسة بدون تقطيع

## الملفات المعدلة
- `web-app/src/constants/api.ts` - إصلاح دالة `requestFreeStream`

## الاختبار
1. افتح تطبيق الويب على Railway
2. اختر أي قناة من صفحة "البث المباشر"
3. يجب أن يبدأ البث فوراً بدون علامة تحميل مستمرة

## ملاحظات
- السيرفر يستخدم HLS Proxy مع segment caching
- اتصال واحد بـ IPTV، عدد غير محدود من المشاهدين
- لا ضغط على حساب IPTV

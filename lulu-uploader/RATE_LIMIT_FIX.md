# إصلاح مشكلة Rate Limit في رافع IPTV

## 🐛 المشكلة

السكريبت كان يحاول إنشاء 60+ مجلد دفعة واحدة قبل البدء بالرفع، مما يسبب:
```
خطأ: LuluStream API error: Requests limit reached: 60 per min
```

## ✅ الحل المطبق

### 1. تأخير عند إنشاء المجلدات
```javascript
// قبل: إنشاء فوري بدون تأخير
const res = await lulu.createFolder(safeName, parentId);

// بعد: تأخير 1.2 ثانية قبل كل مجلد جديد
await xtream.sleep(1200);
const res = await lulu.createFolder(safeName, parentId);
```

**النتيجة**: ~50 مجلد/دقيقة (ضمن حد الـ 60)

### 2. تأخير أطول بعد الرفع
```javascript
// قبل: 3-7 ثواني
await xtream.sleep(3000 + Math.floor(Math.random() * 4000));

// بعد: 5-8 ثواني
await xtream.sleep(5000 + Math.floor(Math.random() * 3000));
```

**النتيجة**: معدل رفع أبطأ وأكثر أماناً

## 📊 معدلات الطلبات الجديدة

### إنشاء المجلدات
- **قبل**: 60+ طلب/دقيقة ❌
- **بعد**: ~50 طلب/دقيقة ✅

### رفع الفيديوهات
- **قبل**: ~10-15 فيديو/دقيقة
- **بعد**: ~8-10 فيديو/دقيقة ✅

## 🔄 سير العمل الجديد

### لكل فيديو:
1. **إنشاء/جلب المجلد** (1.2s تأخير إذا جديد)
2. **رفع الفيديو** (يعتمد على حجم الملف)
3. **حفظ في DB** (فوري)
4. **تأخير 5-8 ثواني** قبل الفيديو التالي

### مثال: رفع 10 أفلام
```
الفيلم 1:
  [📁 جديد] "[AR] Arabic Movies"  (1.2s)
  [📁 جديد] "١١:١١"                (1.2s)
  [⬆ رفع] ١١:١١                    (30-120s حسب الحجم)
  [DB] حفظ البيانات                (0.5s)
  [⏳] تأخير                        (5-8s)

الفيلم 2:
  [📁 موجود] "[AR] Arabic Movies"  (0s - من الكاش)
  [📁 جديد] "12:21"                (1.2s)
  [⬆ رفع] 12:21                    (30-120s)
  [DB] حفظ البيانات                (0.5s)
  [⏳] تأخير                        (5-8s)

... وهكذا
```

## 🎯 الفوائد

### ✅ لا مزيد من أخطاء Rate Limit
- التأخيرات تضمن البقاء ضمن الحدود
- الكاش يقلل الطلبات المكررة

### ✅ معالجة تسلسلية
- كل فيديو يُعالج بالكامل قبل التالي
- البيانات تُحفظ فوراً في DB
- لا فقدان للبيانات عند الانقطاع

### ✅ استئناف تلقائي
- الحالة محفوظة في `iptv-state.json`
- الفيديوهات المرفوعة تُتخطى تلقائياً
- يمكن إيقاف وإعادة التشغيل بأمان

## 📝 الاستخدام

### اختبار بدون رفع
```bash
node iptv.js --dry --mode movies --limit 5
```

### رفع فعلي
```bash
# رفع 10 أفلام
node iptv.js --mode movies --limit 10

# رفع 5 مسلسلات
node iptv.js --mode series --limit 5

# رفع من تصنيف محدد
node iptv.js --mode movies --category "مترجم" --limit 20
```

### مراقبة التقدم
```bash
# عرض الحالة
cat iptv-state.json | jq '.uploaded | length'

# عرض النتائج
cat iptv-results.json | jq 'length'

# عرض الفاشلة
cat iptv-state.json | jq '.failed'
```

## ⚠️ ملاحظات مهمة

### 1. الصبر مطلوب
- رفع 10 أفلام قد يستغرق 10-30 دقيقة
- هذا طبيعي ومقصود لتجنب الحدود

### 2. لا تقاطع العملية
- دع السكريبت يكمل بشكل طبيعي
- إذا اضطررت للإيقاف، استخدم Ctrl+C
- الحالة محفوظة ويمكن الاستئناف

### 3. حدود Lulu API
- **60 طلب/دقيقة** - إنشاء مجلدات
- **5000 طلب/يوم** - جميع العمليات
- السكريبت يحترم هذه الحدود تلقائياً

## 🔍 استكشاف الأخطاء

### خطأ "60 per min"
```bash
# السبب: تجاوز حد الدقيقة
# الحل: انتظر دقيقة وأعد التشغيل
sleep 65 && node iptv.js --mode movies --limit 10
```

### خطأ "5000 per day"
```bash
# السبب: تجاوز الحد اليومي
# الحل: انتظر حتى الغد (إعادة تعيين الحد)
```

### الفيديو لا يُرفع
```bash
# تحقق من السجلات
cat iptv-state.json | jq '.failed'

# أعد المحاولة للفاشلة فقط
# (حذف من .uploaded لإعادة المحاولة)
```

## 📊 الإحصائيات المتوقعة

### رفع 100 فيلم
- **الوقت**: 2-5 ساعات
- **طلبات API**: ~300-500
- **حجم DB**: ~50-100 KB

### رفع 50 مسلسل (10 حلقات لكل)
- **الوقت**: 10-20 ساعة
- **طلبات API**: ~1500-2000
- **حجم DB**: ~200-500 KB

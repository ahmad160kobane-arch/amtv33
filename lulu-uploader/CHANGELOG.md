# سجل التغييرات - IPTV Uploader

## الإصدار 1.1.0 (2026-04-21)

### 🐛 إصلاحات

#### 1. إصلاح Rate Limit (60 req/min)
**المشكلة**: السكريبت كان يحاول إنشاء 60+ مجلد دفعة واحدة، مما يسبب:
```
خطأ: LuluStream API error: Requests limit reached: 60 per min
```

**الحل**:
- ✅ نقل إنشاء المجلدات إلى **بعد** فحص `dryRun`
- ✅ إضافة تأخير 1.2 ثانية قبل كل مجلد جديد
- ✅ زيادة التأخير بعد الرفع من 3-7s إلى 5-8s

**النتيجة**:
```javascript
// قبل: إنشاء مجلدات حتى في وضع --dry
const rootId = await dir(catName);
const fldId  = await dir(title, rootId);
if (dryRun) return 'dry';

// بعد: إنشاء مجلدات فقط عند الرفع الفعلي
if (dryRun) return 'dry';
const rootId = await dir(catName);  // مع تأخير 1.2s
const fldId  = await dir(title, rootId);
```

#### 2. إصلاح MaxListenersExceededWarning
**المشكلة**: تحذيرات مزعجة:
```
MaxListenersExceededWarning: Possible EventEmitter memory leak detected
```

**الحل**:
```javascript
// إضافة في بداية الملف
require('events').EventEmitter.defaultMaxListeners = 20;
```

```
  ✓ إنشاء مجلد الفيلم (1.2s)
  ✓ رفع الفيديو (30-120s)
  ✓ حفظ في DB (0.5s)
  ✓ تأخير (5-8s)
```

### 🎯 الفوائد

1. **✅ لا مزيد من أخطاء Rate Limit**
   - التأخيرات تضمن البقاء ضمن حد الـ 60 req/min
   - الكاش يقلل الطلبات المكررة

2. **✅ وضع الاختبار الحقيقي**
   - `--dry` الآن لا يُنشئ أي مجلدات
   - يمكن اختبار القائمة بدون استهلاك API

3. **✅ معالجة تسلسلية موثوقة**
   - كل فيديو يُعالج بالكامل قبل التالي
   - البيانات تُحفظ فوراً في DB
   - لا فقدان للبيانات عند الانقطاع

4. **✅ لا تحذيرات مزعجة**
   - EventEmitter limit مرفوع إلى 20
   - سجلات نظيفة وواضحة

### 📝 الاستخدام

#### اختبار بدون رفع (لا يُنشئ مجلدات)
```bash
node iptv.js --dry --mode movies --limit 5
```

#### رفع فعلي (مع إنشاء مجلدات وتأخيرات)
```bash
# رفع 10 أفلام
node iptv.js --mode movies --limit 10

# رفع 5 مسلسلات
node iptv.js --mode series --limit 5
```

### ⚠️ ملاحظات مهمة

1. **الصبر مطلوب**
   - رفع 10 أفلام: 10-30 دقيقة
   - رفع 50 فيلم: 1-3 ساعات

2. **حدود Lulu API**
   - 60 طلب/دقيقة (إنشاء مجلدات)
   - 5000 طلب/يوم (جميع العمليات)
   - السكريبت يحترم هذه الحدود تلقائياً

3. **الاستئناف التلقائي**
   - الحالة محفوظة في `iptv-state.json`
   - الفيديوهات المرفوعة تُتخطى تلقائياً
   - يمكن إيقاف وإعادة التشغيل بأمان

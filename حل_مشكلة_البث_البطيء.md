# 🚀 حل مشكلة التأخير في البث المباشر

## 📊 المشكلة

عند فتح قناة البث المباشر، يتأخر البث **5-10 ثواني** قبل أن يبدأ التشغيل.

---

## 🔍 الأسباب المكتشفة

### 1. ⏱️ FFmpeg ينتظر بناء Segments (2-4 ثواني)
- السيرفر ينتظر إنشاء segment واحد على الأقل قبل بدء البث
- كل segment مدته 2 ثانية

### 2. 🔍 فحص حدود الاتصال يضيف تأخير (1-2 ثانية)
- السيرفر ينظف **كل** الجلسات المنتهية قبل كل طلب بث
- يستعلم عن جلسات المستخدم من قاعدة البيانات

### 3. 📡 مزامنة القنوات من Backend (2-5 ثواني)
- إذا القناة غير موجودة في الذاكرة، يسحب **كل القنوات** من backend
- يحدث في أول مرة لكل قناة

**النتيجة:** يوفر **0.5-1 ثانية**

#### 2. إضافة Channel Cache

**الملف:** `cloud-server/server.js` (بعد دالة `_getCachedUser`)

**أضف هذا الكود:**
```javascript
// ─── Channel cache — reduce DB queries ───────────────────
const _channelCache = new Map();
const CHANNEL_CACHE_TTL = 5 * 60 * 1000; // 5 minutes

async function _getCachedChannel(channelId) {
  const cached = _channelCache.get(channelId);
  if (cached && Date.now() - cached.ts < CHANNEL_CACHE_TTL) return cached.data;
  
  const data = await db.prepare(
    'SELECT id, name, stream_url FROM channels WHERE id = ? AND is_enabled = 1'
  ).get(channelId);
  
  if (data) {
    _channelCache.set(channelId, { data, ts: Date.now() });
    // Prevent unbounded cache growth
    if (_channelCache.size > 200) {
      const oldest = _channelCache.keys().next().value;
      _channelCache.delete(oldest);
    }
  }
  
  return data;
}
```

#### 3. استخدام Channel Cache

**الملف:** `cloud-server/server.js` (في `/api/stream/live/:channelId`)

**ابحث عن:**
```javascript
let ch = await db.prepare('SELECT id, name, stream_url FROM channels WHERE id = ? AND is_enabled = 1').get(rawId);
// Lazy sync: if channel not found locally, sync from backend PostgreSQL and retry
if (!ch) {
  await syncChannelsFromBackend(true);
  ch = await db.prepare('SELECT id, name, stream_url FROM channels WHERE id = ? AND is_enabled = 1').get(rawId);
}
```

**استبدله بـ:**
```javascript
let ch = await _getCachedChannel(rawId);
// Lazy sync: if channel not found locally, sync from backend PostgreSQL and retry
if (!ch) {
  await syncChannelsFromBackend(true);
  ch = await _getCachedChannel(rawId);
}
```

**النتيجة:** يوفر **2-5 ثواني** في أول مرة، ثم فوري

### 1. Pre-warming للقنوات الشائعة

**الملف:** `cloud-server/server.js` (في نهاية الملف)

```javascript
// بدء FFmpeg للقنوات الأكثر مشاهدة
const POPULAR_CHANNELS = ['channel_id_1', 'channel_id_2', 'channel_id_3'];

db.init().then(async () => {
  // ... existing code ...
  
  // Pre-warm popular channels
  for (const channelId of POPULAR_CHANNELS) {
    const ch = await _getCachedChannel(channelId);
    if (ch && ch.stream_url) {
      await streamManager.requestStream(channelId, 'live', ch.stream_url, ch.name);
      streamManager.markPermanent(channelId); // لا يتوقف أبداً
      console.log(`[Pre-warm] Started ${ch.name}`);
    }
  }
});
```

**النتيجة:** القنوات الشائعة تبدأ **فوراً** (0 ثانية)

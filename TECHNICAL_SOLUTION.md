# الحل التقني لمشاكل IPTV (511/502/403)

## المشكلة الأساسية
حسابات IPTV غير مستقرة وتفشل بعد فترة قصيرة بسبب:
- HTTP 511: يطلب إعادة المصادقة
- HTTP 502: السيرفر الوسيط لا يستطيع الوصول للمصدر
- HTTP 403: حظر مؤقت بسبب كثرة الطلبات

---

## ✅ الحلول المطبقة حالياً

### 1. إعادة المحاولة الذكية (Smart Retry)
```javascript
// للـ Segments: 5 محاولات بدلاً من 3
for (let attempt = 0; attempt < 5; attempt++) {
  // محاولة التحميل
  if (error === 502 || error === 511) {
    // انتظار تدريجي: 1.5s, 3s, 4.5s, 6s
    await delay(1500 * (attempt + 1));
    continue; // إعادة المحاولة
  }
  if (error === 403) {
    // انتظار 3 ثواني ثم محاولة واحدة فقط
    await delay(3000);
    continue;
  }
}
```

### 2. استخدام الكاش القديم (Stale Cache Fallback)
```javascript
// إذا فشلت كل المحاولات، استخدم الكاش القديم (حتى 6 دقائق)
if (cached && age < SEG_TTL * 3) {
  console.warn('Serving stale cache');
  return cached; // البث يستمر بدون توقف
}
```

### 3. تقليل الضغط على IPTV
```javascript
// إعدادات متوازنة:
MAX_MANIFEST_PARALLEL = 5   // 5 قنوات بالتوازي (ليس 200)
MAX_SEG_PARALLEL = 15        // 15 segment بالتوازي (ليس 500)
MAX_PROACTIVE_PER_CYCLE = 3  // 3 تحديثات استباقية فقط

// تأخير عشوائي بين الطلبات:
await delay(50 + random(100))  // 50-150ms للـ manifest
await delay(20 + random(60))   // 20-80ms للـ segments
```

### 4. Manifest Retry مع تأخير تدريجي
```javascript
// للـ Manifest: 4 محاولات بدلاً من 1
for (let attempt = 0; attempt < 4; attempt++) {
  if (error === 502 || error === 511) {
    // انتظار: 2s, 4s, 6s
    await delay(2000 * (attempt + 1));
    continue;
  }
}
```

## ⚠️ القيود الحالية

### المشكلة الجذرية: **حسابات IPTV غير مستقرة**

**الأعراض:**
- البث يعمل 30 دقيقة ثم يتوقف
- HTTP 511/502/403 تظهر بشكل متكرر
- الحساب يُحظر بعد فترة قصيرة

**السبب:**
1. **جودة مزود IPTV ضعيفة**
   - السيرفرات غير مستقرة
   - حدود استخدام صارمة
   - حظر تلقائي للحسابات النشطة

2. **حدود الحساب الواحد**
   - حساب واحد لا يكفي لبث 24/7
   - المزود يكتشف الاستخدام المكثف
   - يتم حظر الحساب مؤقتاً

```javascript

  { user: 'acc1', pass: 'pass1', priority: 1 },
  { user: 'acc2', pass: 'pass2', priority: 2 },
  { user: 'acc3', pass: 'pass3', priority: 3 }

// عند فشل حساب، التبديل للحساب التالي تلقائياً

  switchTo(account2);
}
```

**المميزات:**
